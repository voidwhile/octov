## 一、产品概述

Octov 是一款跨平台（macOS / Windows）桌面端视频库播放器。支持管理多种存储源中的视频文件，自动识别和分类视频内容，从互联网获取影视元数据（海报、评分、简介等），并提供流畅的播放体验和在线字幕加载。

### 1.1 核心定位

- 一站式视频库管理：本地 + 云盘视频统一管理
- 智能分类：自动识别电影与电视剧
- 精美展示：TMDB 海报和元数据自动刮削
- 流畅播放：支持 HLS 流媒体 + 在线字幕
- 播放记录：自动保存播放进度，支持继续观看

### 1.2 技术栈

| 层级 | 技术 |
|------|------|
| 框架 | Electron 39 + electron-vite |
| 前端 | React 19 + TypeScript 5 |
| 构建 | Vite 7 |
| 播放 | hls.js（HLS 流媒体） |
| API | TMDB v3（元数据）、OpenSubtitles（字幕）、阿里云盘 Open API |

## 四、模块 2：视频扫描与分类

### 4.1 功能说明

系统自动从所有已添加的文件源中递归扫描视频文件，根据文件夹结构和文件名命名规则自动分类为电影或电视剧。

### 4.2 扫描逻辑

1. 遍历每个文件源（文件夹）
2. 递归列出所有文件
3. 筛选视频文件（按文件扩展名或 `category === 'video'`）
4. 按父文件夹分组
5. 对每组应用分类规则

**支持的视频格式**：
`mp4, mkv, avi, mov, wmv, flv, rmvb, ts, m4v, webm`

### 4.3 分类规则

#### 判定为电视剧

满足以下**任一**条件：

1. 同一文件夹下有 **≥ 2 个视频文件**，且文件名包含集号模式
2. 文件名匹配以下集号正则之一：
   - `S\d{1,2}E\d{1,3}` → 如 `S01E01`, `s02e15`
   - `EP?\s*\d{1,3}` → 如 `EP01`, `E01`, `EP 12`
   - `第\s*\d{1,4}\s*[集话期]` → 如 `第1集`, `第01话`, `第100期`
   - 文件夹名包含「季」或 `Season`

#### 判定为电影

不满足电视剧规则的视频文件，判定为电影。

### 4.4 文件名解析

从文件名中智能提取搜索关键词：

```
输入: "星际穿越.Interstellar.2014.BluRay.1080p.mkv"
输出: { title: "星际穿越", englishTitle: "Interstellar", year: 2014 }

输入: "权力的游戏.Game.of.Thrones.S01E01.720p.mkv"
输出: { title: "权力的游戏", season: 1, episode: 1, type: "tvshow" }

输入: "流浪地球2 (2023).mkv"
输出: { title: "流浪地球2", year: 2023 }
```

**解析规则**：
1. 将 `.` 替换为空格
2. 去除分辨率标记（`1080p`, `720p`, `4K`）
3. 去除编码标记（`x264`, `x265`, `HEVC`, `AAC`）
4. 去除来源标记（`BluRay`, `WEB-DL`, `HDRip`）
5. 提取年份 `(19|20)\d{2}`
6. 提取季集号 `S\d+E\d+`
7. 剩余部分作为标题

### 4.5 扫描时机

- 添加/删除文件源时自动触发扫描
- 应用启动时从磁盘缓存加载，后台静默扫描
- 用户手动点击顶部「刷新」按钮时强制重新扫描
- 扫描结果缓存到本地 `media-cache.json`，启动时秒加载

## 五、模块 3：TMDB 元数据刮削

### 5.1 功能说明

使用 TMDB (The Movie Database) API，根据解析出的标题和年份搜索匹配，自动补全以下元数据：

| 字段 | 说明 |
|------|------|
| 中文标题 | TMDB 返回的本地化标题 |
| 原始标题 | 英文原名 |
| 海报图 | `poster_path` → `https://image.tmdb.org/t/p/w500/xxx.jpg` |
| 背景横幅 | `backdrop_path` → `https://image.tmdb.org/t/p/original/xxx.jpg` |
| 评分 | `vote_average`（0-10） |
| 简介 | `overview` |
| 类型标签 | `genres` → 中文标签（科幻、动作、剧情等） |
| 上映日期 | `release_date` / `first_air_date` |
| 时长 | `runtime`（分钟） |
| TMDB ID | 用于后续字幕搜索 |

### 5.2 API Key 配置

- TMDB API Key 通过**设置页面**配置，不硬编码在程序中
- 配置持久化存储在 `userData/app-settings.json`
- 未配置 Key 时，刮削功能静默跳过，不影响基本使用
- 获取方式：注册 [TMDB 账号](https://www.themoviedb.org/signup) → 设置 → API → 申请 Developer Key

### 5.3 匹配策略

1. 优先使用中文标题搜索
2. 无结果时使用英文标题搜索
3. 指定年份缩小范围
4. 取搜索结果第一条（最相关）
5. 搜索失败时使用文件名作为标题，不显示海报

### 5.4 缓存策略

- 以 `文件路径 hash` 为 key 缓存元数据
- 缓存存储在 `userData/tmdb-cache.json`
- 已缓存的文件不再重复请求 TMDB
- 用户可手动点击刷新按钮重新刮削

## 七、模块 5：视频播放器

### 7.1 功能说明

支持播放本地视频文件和云盘 HLS 流媒体视频。

### 7.2 云盘视频播放流程

```
点击播放 → 路由 /player/cloud/{fileId}
  → 调用 aliyunDrive.getVideoPlayInfo(fileId)
  → 获取 m3u8 转码流地址
  → hls.js 加载并播放
  → 恢复上次播放位置（如有记录）
  → 定时同步播放进度到云盘 + 本地记录
```

### 7.3 播放器功能

| 功能 | 说明 |
|------|------|
| 播放/暂停 | 空格键 / 点击画面 |
| 上一集/下一集 | 按文件名排序，同目录视频列表 |
| 进度条 | 拖拽或点击跳转 |
| 快进/快退 | ← → 键，10秒 |
| 音量 | ↑ ↓ 键 |
| 倍速 | 0.5x ~ 2x |
| 全屏 | F 键 |
| 字幕 | 在线加载（见模块 6） |
| 窗口拖拽 | 顶部栏区域支持拖拽移动窗口 |
| 返回 | 回退到进入播放器前的页面 |

### 7.4 播放记录

- 播放开始时立即记录
- 每 15 秒定期更新进度
- 暂停/结束/退出播放器时保存最终进度
- 记录持久化存储在 `userData/play-history.json`
- 最多保留 100 条记录
- 再次播放时自动恢复上次播放位置（进度 < 95% 且 > 5秒）

### 7.5 集切换逻辑

- 上一集/下一集基于同目录下视频文件列表（按文件名排序）
- 切换集数使用 `replace: true`，不增加历史栈
- 返回按钮始终回到打开播放器的页面（非上一集）

### 7.6 HLS 流选择

阿里云盘 `getVideoPreviewPlayInfo` 返回多个转码模板：

| 模板 | 分辨率 |
|------|--------|
| LD | 540p |
| SD | 720p |
| HD | 1080p |
| FHD | 全高清 |

默认选择最高可用清晰度，用户可切换。

## 八、模块 6：在线字幕

### 8.1 功能说明

播放视频时自动从互联网搜索并加载匹配的字幕文件。

### 8.2 字幕搜索策略

搜索参数优先级：
1. **TMDB ID** + 季集号（最精确）
2. **视频文件名**关键词搜索
3. **视频文件 hash**（部分字幕站支持）

### 8.3 字幕格式支持

| 格式 | 扩展名 | 支持 |
|------|--------|------|
| SubRip | .srt | ✅ |
| WebVTT | .vtt | ✅ |
| ASS/SSA | .ass/.ssa | ✅（转换为简单样式） |

## 十、数据结构定义

### 10.1 MediaItem

```typescript
interface MediaItem {
  id: string
  title: string
  originalTitle?: string
  type: 'movie' | 'tvshow' | 'other'
  year?: number
  poster: string           // TMDB 海报 URL
  backdrop?: string        // TMDB 背景图 URL
  rating?: number          // TMDB 评分
  genres?: string[]        // 类型标签
  overview?: string        // 剧情简介
  duration?: number        // 时长（分钟）
  dateAdded: string
  releaseDate?: string
  filePath?: string
  source?: 'local' | 'aliyundrive'
  seasons?: Season[]
  progress?: PlaybackProgress
  cloudFileId?: string     // 阿里云盘文件 ID
  tmdbId?: number          // TMDB ID（用于字幕搜索）
  sourceId?: string        // 所属文件源 ID
}
```

### 10.2 PlayRecord（播放记录）

```typescript
interface PlayRecord {
  fileId: string           // 云盘文件 ID（唯一标识）
  name: string             // 视频名称
  title?: string           // TMDB 标题
  poster?: string          // 海报
  currentTime: number      // 播放位置（秒）
  totalDuration: number    // 总时长（秒）
  percentage: number       // 播放百分比 0-100
  lastPlayed: string       // 最后播放时间
  parentId?: string        // 父目录 ID
}
```

### 10.3 本地持久化

```
userData/
├── aliyundrive-config.json   # 阿里云盘授权 token
├── file-sources.json         # 已添加的文件源列表
├── media-cache.json          # 扫描+TMDB 结果缓存
├── play-history.json         # 播放记录（继续观看）
├── app-settings.json         # 应用设置（TMDB Key 等）
└── tmdb-cache.json           # TMDB 元数据缓存
```

## 十一、非功能性需求

| 项目 | 要求 |
|------|------|
| 性能 | 扫描 1000+ 文件在 10 秒内完成 |
| 启动 | 从磁盘缓存秒加载媒体库，后台按需扫描 |
| 缓存 | TMDB 结果永久缓存，避免重复请求 |
| 离线 | 无网络时使用缓存数据，不影响浏览 |
| 安全 | API Key 存储在主进程配置文件中，通过 IPC 通信 |
| 错误处理 | 网络失败静默降级，不阻断用户操作 |
| 导航 | 所有页面支持正确的返回导航（历史栈回退） |
